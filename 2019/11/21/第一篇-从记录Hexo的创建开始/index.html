前言

​ Hexo 是一个基于 NodeJs 博客框架，可以快速的帮我们搭建一个博客系统，Hexo使用的是Markdown解析文章的，在几秒内即可利用靓丽的主体生成静态网页。 同时可以托管至github，免去服务器要求(如果有条件就随意咯)。

1.安装Hexo

安装 Hexo 相当简单，只需要先安装下列应用程序即可

Node.js

Git

步骤省略，安装两个前置工具后执行命令(使用git Bsah)

$ npm install -g hexo-cli

  • 此时可能会出现的问题：PowerShell:在此系统上禁止运行脚本`
  • 解决步骤:
    • 使用管理员权限打开powershell，进入hexo安装目录，执行get-executionpolicy命令
    • 根据反馈执行set-executionpolicy remotesigned命令，回复y即可 顺利安装hexo。

5.部署到本地

​ Hexo 3.0 把服务器独立成了个别模块，您必须先安装 hexo-server 才能使用。

$ npm install hexo-server --save

​ 安装完成后，输入以下命令以启动服务器，网站会在 http://localhost:4000 下启动。在服务器启动期间，Hexo 会监视文件变动并自动更新，无须重启服务器。

hexo server (hexo s)

​ 本地部署后博客的地址以日期命名

http://localhost:4000/2019/11/21/%E7%AC%AC%E4%B8%80%E7%AF%87-%E4%BB%8E%E8%AE%B0%E5%BD%95Hexo%E7%9A%84%E5%88%9B%E5%BB%BA%E5%BC%80%E5%A7%8B/

在本地部署之后如果内容有更新，只需重新g和s命令即可，如要在远端同步，执行g和d命令

6.托管到github Pages仓库

github pages 集成在 github 中, 直接和代码管理绑定在一起, 随着代码更新自动重新部署, 使用非常方便.
6.1 先创建github Pages仓库

​ 在github中创建repo，其名字为github的username.github.oo 每个账号只有一个，会提供一个免费的二级域名，域名名称即为仓库名称

*roxBear/roxBear.github.io*

6.2 设置Hexo部署ip为自己的githubPages地址

​ 注：此处采用ssh方式连接，可以在修改源文件后自动部署

在hexo的config.ym中配置

6.3 hexo部署

$ hexo g + $ hexo d Hexo部署会将本地源码推至pages仓库

7.建立分支保存源文件

​ 博客的源码都在本地的public文件夹中，如果丢失或者想在其他电脑上修改博客，则无法实现。因为这个博客目录并没有从pages仓库中clone。也就是说这个博客的代码托管是单向的，所以需要将pages仓库的master分支clone到其他目录

(hexo托管在master分支)

​ 注意，如果使用了其它的主题，可以删除主题文件夹中的 .git 文件，以免主题文件夹下的文件不能加入暂存区，不然以后clone下来的文件没有包括主题，hexo s会出现hexo no layout index.html
